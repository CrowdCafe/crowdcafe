F.A.Q.

I have pulled the Startup framework from the repository and its components are not displaying properly. What does this mean?

The Startup Framework is based on the Flat-UI framework and its repository contains Flat-UI as a project submodule. The above mentioned situation can happen if you have pulled the project without the submodule. In order to fix this, you need to execute the following commands in your git console:

git submodule init
git submodule update

After having done this, the submodule folder 'flat-ui' shouldn't be empty, and Startup components will display correctly.

Is Startup framework has some specific CSS classes or it only uses classes from its based frameworks?

Startup framework is based on Flat-UI 2.0.0, but it has owned CSS classes and LESS mixins.
You can watch it in the special Memo.

We recomend save this page if you will work with our framework.

I want to use the 'less' version. How can I do it?

LESS version it the same version of the "STARTUP framework".

In order to develop with "LESS", you should already have set up your development environment for this. You can read more about it on the official site of the "LESS framework".

Could I use the "STARTUP framework" without 'LESS'?

Yes. You can use the "STARTUP framework" using simple version instructions.

Can I use bootstrap elements inside your framework.

"STARTUP framework" based on bootstrap. So you can use boostrap elements inside it.

What version of "Bootstrap" the "STARTUP framework" use as base?

"STARTUP framework" based on Bootstrap v2.3.2.

Can I use Bootstrap 3 inside the "STARTUP framework"?

"STARTUP framework" until that is not compatible with 3 version.
